# B站内容创作

## 文案风格

### 核心调性：知识+趣味、弹幕文化、一键三连

- **知识性**：B站用户偏好有深度、有干货的内容
- **趣味性**：严肃内容也要有趣味包装（梗、比喻、动画）
- **弹幕友好**：留出弹幕互动空间（提问、投票、玩梗）
- **社区感**：用 UP 主 - 观众的关系，不是主播 - 粉丝
- **二次元素**：适当使用 B 站特色用语和文化

### B 站特色用语

```
打招呼: "各位观众老爷好" / "大家好我是{名字}"
求三连: "如果觉得有用，一键三连支持一下"
投币: "币都投了吧" / "硬币给你了"
弹幕互动: "弹幕告诉我..." / "弹幕刷'懂了'让我看看"
分P: "详细教程在P2" / "这期先讲{部分}，下期讲{部分}"
预告: "下期预告：{主题}，点个关注不迷路"
```

### 视频文案结构

```
开场（0-15s）:
  - 标题呼应（"今天聊聊{话题}"）
  - 简短Hook（问题/成果/反常识）
  - 内容大纲预告

正文（15s-核心内容）:
  - 分段清晰，每段有小标题
  - 穿插弹幕互动点
  - 用类比/梗/动画辅助理解
  - 关键信息重复+文字标注

总结（最后15-30s）:
  - 核心要点回顾
  - 一键三连引导
  - 下期预告
```

## 格式规范

### 中视频（B站核心形态）

| 项目 | 规范 |
|------|------|
| 比例 | 横屏 16:9（1920×1080）推荐 |
| 时长 | 3-10 分钟最佳 |
| 格式 | MP4 / FLV / AVI |
| 大小 | ≤8GB |
| 帧率 | 30fps / 60fps |
| 编码 | H.264 / H.265 |
| 码率 | ≥6Mbps（1080p） |

### 短视频（竖屏模式）

| 项目 | 规范 |
|------|------|
| 比例 | 竖屏 9:16（1080×1920） |
| 时长 | ≤3 分钟 |
| 入口 | B站短视频 Tab |
| 适用 | 快节奏、碎片化内容 |

### 封面规范

```
尺寸: 1920×1080（16:9）
文件: JPG/PNG, ≤5MB

要素:
  - 大字标题（≤12字，加描边/阴影）
  - 关键人物/画面
  - 分区标识色（可选）
  - 系列编号（如有）

B站封面特点:
  - 可以比其他平台信息量更大
  - 文字密度更高（B站用户习惯读封面）
  - 可用"【】"标注分类
```

### 字幕规范

```
必须有字幕（B站用户在各种场景看视频）
位置: 画面下方，不遮挡主内容
字体: 黑体，白色+黑色描边
关键词: 用彩色高亮
术语: 首次出现标注英文原文
```

## 标题规则

### 基本规范

| 规范 | 说明 |
|------|------|
| 字数 | ≤80 字（B站允许长标题！这是优势） |
| 关键词 | 前 20 字包含核心关键词 |
| 格式 | 【分类】+ 主标题 + 副标题 |
| 特点 | 信息量可以大，不怕长 |

### 标题公式

```
【分类】+ 核心内容 + 价值点
  "【保姆级教程】从零搭建个人博客，小白也能30分钟搞定"

数字 + 主题 + 受众
  "2024年程序员必看的10个AI工具，效率直接翻倍"

反差/悬念 + 具体
  "实习生用AI写的代码，让老程序员沉默了"

系列 + 编号
  "【AI全栈开发】第3期：用Claude API搭建智能客服"
```

### 【】标签格式（B站特色）

```
常用:
  【教程】【测评】【科普】【避坑】【保姆级】
  【实测】【对比】【盘点】【合集】【干货】

领域:
  【前端】【Python】【AI】【设计】【运营】
  【游戏】【数码】【美食】【生活】
```

## 分区选择

### 主要分区（影响推荐）

| 分区 | 适用内容 | 竞争度 |
|------|----------|--------|
| 科技 > 软件应用 | 工具教程、App测评 | 中 |
| 科技 > 计算机技术 | 编程、开发、技术 | 中 |
| 知识 > 社科·法律·心理 | 商业分析、行业解读 | 低 |
| 知识 > 职业职场 | 职场技能、副业 | 中 |
| 生活 > 日常 | vlog、生活记录 | 高 |
| 影视 > 影视杂谈 | 影视解说 | 高 |

### 分区选择策略

```
原则:
  1. 选最匹配的分区（不要为了流量选不匹配的分区）
  2. 分区越垂直，推荐越精准
  3. 同一账号尽量固定1-2个分区
  4. 新分区/冷门分区竞争小，更容易起量
```

## 发布时间

### 最佳时段

| 时段 | 流量级别 | 适合内容 |
|------|----------|----------|
| 10:00-12:00 | ★★★ | 知识/教程类 |
| 12:00-14:00 | ★★★★ | 午休刷B站 |
| 17:00-19:00 | ★★★★ | 下班/放学 |
| 20:00-23:00 | ★★★★★ | 黄金时段 |
| 周末全天 | ★★★★ | 整体流量高于工作日 |

### 发布频率

```
推荐: 每周 2-3 条（质量 > 数量）
B站用户对质量要求高，1条精品 > 5条凑数
保持更新频率稳定，培养用户期待
```

## 审核规则

详见 → `references/content-rules.md`

### B站审核特点

```
特殊严格项:
  1. 转载必须标注来源（转载 vs 自制）
  2. 原创标记错误会被处罚
  3. 版权规则严格（音乐、影视素材）
  4. 弹幕/评论区也会审核
  5. 涉政内容零容忍

审核流程:
  提交 → 机审（通常几分钟）
       → 人审（高流量或被举报时）
       → 定期巡查（已发布内容）
```

### 稿件分类标注

```
必选:
  - 自制/转载（错误标注会被处罚）
  - 分区选择
  - 标签

可选但推荐:
  - 合作/广告声明（恰饭必须标注）
  - 付费推广标记
  - 内容警告（如有）
```

### 恰饭（商业合作）规则

```
必须:
  - 标注"本视频包含商业推广"
  - 使用B站花火平台（官方合作通道）

建议:
  - 恰饭内容也要有价值（不能纯广告）
  - 提前在视频开头声明
  - 保持对产品的客观评价（B站用户反感硬广）
```

## 算法适配

详见 → `references/algorithm-guide.md`

### 核心指标

```
权重排序:
  1. 播放量            — 基础指标
  2. 弹幕数            — B站特色高权重指标
  3. 投币数            — 用户认可度（需消耗硬币）
  4. 收藏数            — 内容长期价值
  5. 分享数            — 外部传播力
  6. 点赞数            — 基础互动
  7. 完播率            — 内容质量
  8. 评论数/质量        — 社区活跃度
```

### 热门推荐机制

```
首页推荐:
  - 基于用户兴趣标签匹配
  - 视频质量分（互动数据综合）
  - UP主权重（粉丝数、活跃度、历史数据）

分区热门:
  - 24小时内分区综合排名
  - 数据增速 > 绝对数值
  - 小UP主也有机会上热门

每周必看:
  - 编辑推荐
  - 极高质量要求
  - UP主影响力+内容质量
```

### 一键三连引导策略

```
自然引导（推荐）:
  "如果这期内容对你有帮助，三连支持一下"

趣味引导:
  "据说三连的人{美好祝愿}"
  "投币的都是大帅哥大美女"

互惠引导:
  "三连过{数字}我就出{主题}的教程"
  "你们的每一个三连都是我肝视频的动力"

避免:
  "不三连的都是{负面}"（引起反感）
  过度频繁地要三连
```
