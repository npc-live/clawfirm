# Twitter/X 推荐算法

## For You 推荐机制

### 推荐来源

```
For You 时间线组成:
  50% — 关注的人的推文
  50% — 算法推荐的推文

推荐候选来源:
  1. 你关注的人的推文
  2. 你关注的人互动过的推文
  3. 与你兴趣相似的用户的推文
  4. 热门/趋势内容
```

### 推荐因子权重

```
正面因子（按权重排序）:
  1. 回复（Reply）        ★★★★★  (+54x)
  2. 引用转推（Quote RT）  ★★★★   (+20x)
  3. 转推（Retweet）      ★★★    (+4x)
  4. 点赞（Like）         ★★     (+1x, 基准)
  5. 长停留时间           ★★     (阅读时间)
  6. 点击展开全文         ★★     (兴趣信号)
  7. 个人主页访问         ★★     (关注意图)
  8. 点击链接             ★      (外部兴趣)

负面因子:
  - Mute（-74x）
  - Block（极大负面）
  - Report（极大负面）
  - "Not Interested"（负面信号）
```

### 关键洞察

```
1. 回复 >> 一切
   引导用户回复比引导RT更有价值

2. 引用 > 转推
   鼓励引用转推而非简单RT

3. 停留时间很重要
   长文/Thread/图片 → 增加停留时间

4. 被Mute比被忽略更糟
   不要发太多无价值内容（被Mute -74x）
```

## 蓝标（Premium）权重

### 蓝标优势

```
推荐权重:
  - 蓝标用户推文在 For You 中权重更高
  - 蓝标回复在评论区排序更靠前
  - 蓝标用户的投票权重更高

功能解锁:
  - 长推文（>280字符）
  - 更长的视频上传
  - 编辑推文
  - 推文书签文件夹
  - 减少广告
```

## 内容类型偏好

### 算法偏好的内容

```
高推荐概率:
  + 有图片/视频的推文（比纯文字+40%曝光）
  + Thread（增加停留时间）
  + 引发大量回复的推文
  + 原创内容（非RT）
  + 蓝标用户内容

低推荐概率:
  - 纯链接推文（降权明显）
  - 频繁的自动转推
  - 重复内容
  - 仅含hashtag的推文
```

### Thread 算法优势

```
Thread 为什么受算法偏好:
  1. 增加停留时间（逐条阅读）
  2. 多次曝光机会（每条都可能被推荐）
  3. 引发更多回复（每条都可以回复）
  4. 书签/收藏率高
  5. 引用转推率高（容易引用单条）
```

## 时间衰减

```
推文时效性:
  - 推文的推荐权重随时间衰减
  - 黄金期: 发布后 15-60 分钟
  - 活跃期: 发布后 1-6 小时
  - 长尾期: 6-24 小时（被RT/引用可延长）
  - 过期: >24小时（除非持续获得互动）

策略:
  - 在粉丝最活跃时段发布
  - 发布后 30 分钟内积极互动回复
  - 自评论补充内容（保持活跃度）
```

## 账号信誉分

```
影响账号推荐权重的因素:

正面:
  + 账号年龄（老账号权重高）
  + 粉丝质量（真实活跃粉丝 > 僵尸粉）
  + 持续活跃
  + 低被举报率
  + 蓝标认证

负面:
  - 频繁被举报
  - 被标记为spam
  - 关注/粉丝比异常
  - 大量粉丝是新建/不活跃账号
  - 违规记录
```

## 互动优化策略

### 引导回复（最高权重）

```
技巧:
  - 末尾提问: "What's your take?"
  - 争议观点: 引发讨论
  - 投票: 用Twitter投票功能
  - 填空: "The best tool for X is ___"
  - 排名: "Rank these: A, B, C, D"
```

### 引导引用转推

```
技巧:
  - 提供可引用的金句
  - "Add your take and RT"
  - 创造可加评论的框架
  - 发起 #话题 挑战
```

### 增加停留时间

```
技巧:
  - 使用Thread（逐条阅读）
  - 附带图片/信息图
  - 长文式推文（蓝标用户）
  - 数据密集内容
```
