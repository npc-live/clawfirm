# Twitter/X 格式规范

## 推文文字

| 项目 | 规范 |
|------|------|
| 字符限制 | 280 字符（普通）/ 25,000 字符（蓝标） |
| 中文计算 | 1 中文字 = 2 字符 |
| 链接长度 | 固定 23 字符（t.co 缩短） |
| 换行 | 支持，用于格式化 |
| @mention | 不占额外字符 |

---

## 图片规范

### 单图/多图

| 项目 | 规范 |
|------|------|
| 数量 | 最多 4 张/推 |
| 格式 | JPG, PNG, GIF, WEBP |
| 大小 | ≤5MB（图片）/ ≤15MB（GIF） |
| 分辨率 | 推荐 1600×900 或 1200×675 |

### 图片尺寸推荐

| 场景 | 比例 | 尺寸 |
|------|------|------|
| 单图 | 16:9 | 1600×900 |
| 双图 | 7:8 each | 700×800 |
| 三图 | 7:8 + 16:9 | 混合 |
| 四图 | 2:1 each | 1200×600 |
| 信息图 | 2:1 | 1600×800 |

### 图片最佳实践

```
推荐:
  - 高对比度（手机端清晰）
  - 文字大号（可在feed中直接阅读）
  - 品牌色统一
  - Alt text 填写（SEO + 无障碍）

避免:
  - 低分辨率/模糊
  - 文字过小
  - 敏感内容未标记
```

### Alt Text

```
重要: 为每张图片添加 Alt text
  - 提高搜索可见性
  - 无障碍（屏幕阅读器）
  - 算法参考（理解图片内容）

字数限制: 1000 字符
写法: 简洁描述图片内容和关键信息
```

## 视频规范

### 视频参数

| 项目 | 规范 |
|------|------|
| 时长 | ≤2:20（普通）/ ≤60min（蓝标）/ ≤4h（Premium+） |
| 格式 | MP4（H264+AAC） |
| 大小 | ≤512MB |
| 分辨率 | 1920×1080（推荐）/ 最高 4096×4096 |
| 帧率 | 30-60fps |
| 码率 | ≤25Mbps |
| 比例 | 16:9（横屏）/ 1:1 / 9:16（竖屏） |

### 视频缩略图

```
自动: 从视频中选取一帧
自定义: 暂不支持自定义缩略图（需在视频中设计好首帧）

技巧:
  - 视频第一帧就是"封面"
  - 确保第一帧有吸引力
  - 可在视频开头插入"标题帧"
```

## Thread 格式

### 编号规范

```
方式1: 数字/总数
  "1/10"
  "2/10"
  ...

方式2: 仅数字
  "1/"
  "2/"
  ...

方式3: emoji + 数字
  "🧵 1/"
  "🧵 2/"
  ...
```

### Thread 结构建议

```
条数: 5-12 条最佳
每条: 独立可读的完整想法
首推: 最强Hook（决定Thread命运）
末推: CTA + 自引RT
间隔: 每条间无需等待，可连续发布
```

## Poll（投票）规范

| 项目 | 规范 |
|------|------|
| 选项数 | 2-4 个 |
| 选项长度 | ≤25 字符/选项 |
| 时长 | 5分钟 - 7天 |
| 默认时长 | 24小时 |

## Spaces（语音空间）

```
标题: ≤100 字符
描述: ≤280 字符
时长: 无限制
录制: 可选（蓝标功能）
```

---

## 链接预览

```
当推文含链接时:
  - 自动生成预览卡片
  - 显示: 标题 + 描述 + 图片
  - 链接固定占 23 字符

优化预览:
  - 确保目标页面有 og:title, og:description, og:image
  - og:image 推荐 1200×628（1.91:1）
```

---

## 排版建议

### 单推排版

```
好的排版:
  [Hook一句话]

  [核心内容，每点换行]

  [CTA]

  #Tag1 #Tag2

避免:
  一大段不分段的文字
  过多标点符号
  全大写
```

### 使用换行增加可读性

```
技巧:
  - 用空行分隔不同信息
  - 用 emoji 作为列表标记
  - 短句 > 长句
  - 一条推文一个核心信息
```
