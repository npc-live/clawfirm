# Twitter/X 内容创作

## 文案风格

### 核心调性：简洁犀利、观点驱动

- **简洁**：一句话能说清就不用两句
- **观点鲜明**：有立场，不做"正确的废话"
- **对话感**：像和一个聪明人 1:1 聊天
- **信息密度高**：每句都要有信息量
- **英文为主 or 中英混搭**（取决于目标受众）

### 语气参考

```
观点型: "Hot take: {观点}"
分享型: "TIL: {今日所学}"
教程型: "How to {目标} (a thread) 🧵"
反思型: "I used to think {旧认知}. Now I know {新认知}."
数据型: "{数据}. Let that sink in."
故事型: "A year ago, I {起点}. Today, {终点}. Here's what I learned:"
```

## 格式规范

### 单推（≤280 字符）

| 项目 | 规范 |
|------|------|
| 字符限制 | 280 字符（中文约 140 字） |
| 换行 | 用于节奏控制，每个要点一行 |
| 链接 | 占 23 字符（t.co 缩短） |
| 图片 | 最多 4 张，不占字符 |
| 视频 | 最长 2:20（普通）/ 长视频需要认证 |

### Thread（5-12 条）

| 项目 | 规范 |
|------|------|
| 条数 | 5-12 条最佳（太短无深度，太长看不完） |
| 首推 | = Hook，决定整个 Thread 的命运 |
| 编号 | "1/" 或 "🧵 1/10" 格式 |
| 末推 | = CTA + 自引 RT（"If you found this useful, RT the first tweet"） |
| 节奏 | 每条一个要点，不要跨推分拆一个想法 |

### Thread 结构模板

```
1/ [Hook — 最强一句话，抓注意力]

2/ [背景/问题 — 为什么这个话题重要]

3/-9/ [核心内容 — 每条一个要点]
  - 每条独立可读
  - 用数字/emoji/换行增加可读性
  - 穿插数据、案例、截图

10/ [总结 — 简洁回顾]

11/ [CTA — "Follow @xxx for more" + "RT the first tweet"]
```

### Quote RT

比普通 RT 更有利于算法推荐：
- 添加你自己的观点/评论
- 补充上下文或数据
- 提出不同视角

## 标题规则（首推 = 标题）

### 首推公式

| 公式 | 示例 |
|------|------|
| 成果 + How | "I grew from 0 to 50K followers in 6 months. Here's exactly how 🧵" |
| 数字清单 | "10 tools I use daily that save me 5+ hours/week:" |
| 反常识 | "Unpopular opinion: {观点}" |
| 痛点 | "Most people waste hours on {X}. There's a better way:" |
| 权威 | "After {N} years in {field}, here are {N} things I wish I knew:" |

### 首推检查清单

- [ ] 是否在第一句就传达了核心价值？
- [ ] 是否给了继续读下去的理由？
- [ ] 是否能独立成为一条好推文？（不依赖后续）
- [ ] 是否包含具体数字？
- [ ] 是否避免了点击诱饵（clickbait）感？

## 标签策略

### 基本规则

- ≤3 个 hashtag（过多降权）
- 行业/领域标签为主
- 不要在正文中间插 hashtag（影响阅读体验）
- 放在推文末尾或第一条评论

### 标签选择

```
推荐:
  #BuildInPublic  #IndieHacker  #AI  #SaaS  #Startup
  #WebDev  #ProductHunt  #NoCode  #CreatorEconomy

避免:
  #FollowForFollow  #Like4Like（spam 标记）
  过度细分的标签（无人搜索）
  与内容无关的热门标签（降权）
```

## 发布时间

### 目标受众为全球/北美

| 时段（UTC-8 PST） | 北京时间 | 流量级别 |
|-------------------|----------|----------|
| 8:00-10:00 AM | 00:00-02:00 | ★★★★★ |
| 12:00-1:00 PM | 04:00-05:00 | ★★★★ |
| 5:00-6:00 PM | 09:00-10:00 | ★★★★ |

### 目标受众为亚太/中文圈

| 北京时间 | 说明 | 流量级别 |
|----------|------|----------|
| 8:00-9:00 | 早通勤 | ★★★ |
| 12:00-13:00 | 午休 | ★★★★ |
| 21:00-23:00 | 北美早间 + 亚太晚间重叠 | ★★★★★ |

### 发布频率

- 日常：3-5 条原创/互动
- Thread：每周 1-2 个
- 保持活跃度（算法偏好持续活跃的账号）

## 风控规则

详见 → `references/content-rules.md`

### 关键风控

| 风险行为 | 后果 |
|----------|------|
| hashtag 过多（>5） | 降权/spam 标记 |
| 短时间大量发推 | 限流/临时锁定 |
| 重复内容（复制粘贴） | 降权 |
| 链接 spam（每条都带链接） | 降权 |
| 自动化行为（批量关注/取关） | 封号风险 |
| 敏感内容无标记 | 限流/封号 |

### Bot 判定规避

```
避免:
  - 固定间隔发推（看起来像定时器）
  - 完全一致的推文格式
  - 只发不互动
  - 短时间大量关注/取关

建议:
  - 发推间隔随机化
  - 混合原创+互动+RT
  - 保持真实的互动模式
  - 手动操作 > 自动化
```

## 算法适配

详见 → `references/algorithm-guide.md`

### For You 推荐因子

```
权重排序:
  1. 回复（Reply）    — 最高权重
  2. 引用转推（Quote） — 高权重
  3. 转推（Retweet）  — 中等权重
  4. 点赞（Like）     — 基础权重
  5. 长停留时间       — 加分
  6. 点击展开全文     — 加分
  7. 个人主页访问     — 加分
```

### 算法友好策略

| 策略 | 说明 |
|------|------|
| 鼓励回复 | 末尾提问 / 争议话题 / "What do you think?" |
| 鼓励引用 | 提供可引用的金句/观点 |
| 增加停留 | Thread / 长图 / 信息密集内容 |
| 减少跳出 | 避免纯链接（link-only tweets 降权） |
| 互动回馈 | 回复你的评论区（算法偏好互动活跃的账号） |

### 蓝标（Premium）权重

- 蓝标用户的内容在 For You 中权重更高
- 蓝标用户的回复排序更靠前
- 蓝标解锁长推文（>280 字符）和更长视频
