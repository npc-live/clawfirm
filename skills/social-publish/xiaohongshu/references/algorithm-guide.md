# 小红书推荐算法

## CES 评分模型

CES（Community Engagement Score）是小红书衡量内容质量的核心指标。

### 计算公式

```
CES = 点赞数×1 + 收藏数×1 + 评论数×4 + 转发数×4 + 关注数×8
```

### 权重解读

| 行为 | 权重 | 含义 |
|------|------|------|
| 点赞 | ×1 | 基础认可 |
| 收藏 | ×1 | 实用价值 |
| 评论 | ×4 | 深度互动 |
| 转发 | ×4 | 传播价值 |
| 关注 | ×8 | 最高认可 |

### 优化重点

```
高效策略（按ROI排序）:
  1. 引导关注（×8）: 系列内容、人设建设、预告下期
  2. 引导评论（×4）: 提问、投票、争议话题
  3. 引导转发（×4）: 实用清单、工具推荐
  4. 引导收藏（×1）: 教程、模板、指南
  5. 引导点赞（×1）: 情感共鸣、优质封面
```

## 流量池机制

### 分级流量池

```
第1级 - 初始曝光池（200-500）
  ├── 发布后即时分发
  ├── 主要给粉丝+少量相似用户
  └── 判定: 2小时内 CES 表现

第2级 - 扩大池（2000-5000）
  ├── CES 达标后进入
  ├── 扩大到更多兴趣相似用户
  └── 判定: 24小时内数据表现

第3级 - 发现页（1万-5万）
  ├── 出现在发现页/搜索结果
  ├── 内容标签精准匹配
  └── 判定: 数据持续增长

第4级 - 热门池（10万+）
  ├── 大范围分发
  ├── 可能出现在热搜/热门话题
  └── 通常需要持续 24-72 小时的良好数据

第5级 - 长尾池（持续流量）
  ├── 搜索流量持续导入
  ├── "常青"内容可持续获取流量数月
  └── 取决于 SEO 关键词匹配度
```

### 流量池升级条件

| 池级 | 关键指标 | 参考阈值 |
|------|----------|----------|
| 1→2 | 互动率 | > 5%（互动数/曝光数） |
| 2→3 | 完读率+互动率 | 完读 > 40%，互动 > 3% |
| 3→4 | 综合数据 | 持续增长，无下降趋势 |
| 4→5 | 搜索匹配 | 关键词排名靠前 |

## 标签权重

### 标签对推荐的影响

```
标签作用:
  1. 定义内容主题 → 匹配目标用户
  2. 影响搜索排名 → 长尾流量
  3. 关联话题活动 → 额外曝光

标签策略:
  - 核心标签（2个）: 精准匹配，搜索量中等
  - 长尾标签（3个）: 竞争小，转化率高
  - 热门标签（1个）: 蹭流量，相关性要有
```

### 搜索 SEO 优化

```
影响搜索排名的因素:
  1. 标题关键词匹配度（最高权重）
  2. 正文关键词密度（自然嵌入，不要堆砌）
  3. 标签匹配度
  4. 笔记互动数据（CES）
  5. 账号权重（粉丝数、活跃度）
  6. 内容时效性（近期发布的优先）
```

## 内容生命周期

```
阶段1 - 爆发期（0-24h）
  最重要的数据积累窗口
  互动数据决定是否进入下一级流量池

阶段2 - 增长期（1-7天）
  如果数据好，持续获得推荐
  搜索流量开始贡献

阶段3 - 长尾期（7天+）
  主要靠搜索流量
  "常青内容"可持续数月
  关键词排名稳定后流量趋于稳定

小红书的特点: 长尾效应强
  - 好内容可以在发布数月后仍有流量
  - 搜索是重要的流量来源
  - 所以 SEO 优化比其他平台更重要
```

## 账号权重

### 影响账号权重的因素

```
正面因素:
  + 完善的账号信息
  + 稳定的发布频率
  + 持续的高互动率
  + 垂直领域内容（不跨领域太多）
  + 原创内容占比高
  + 平台认证

负面因素:
  - 违规记录
  - 长期不活跃
  - 内容领域混乱
  - 搬运/低质量内容多
  - 异常互动行为（刷量）
```
