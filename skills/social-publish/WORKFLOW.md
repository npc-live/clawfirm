# 自媒体内容发布工作流

## 概览

从选题到发布的完整自媒体内容创作流水线。采用"平台为主 + 通用基座"的混合架构。

```
┌─────────────────────────────────────────────────────────────────┐
│                     输入：选题 / 产品 / 热点                      │
└─────────────────────────────────┬───────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────┐
│              Step 1: copywriting-base                            │
│              通用文案策略 + 标题公式 + CTA 模板                    │
└─────────────────────────────────┬───────────────────────────────┘
                                  ↓
                    ┌─────────────┼─────────────┐
                    ↓             ↓             ↓
            ┌───────────┐ ┌───────────┐ ┌───────────┐
            │ 小红书     │ │ Twitter   │ │ 抖音/B站   │
            │ 图文为主   │ │ Thread    │ │ 短视频     │
            └─────┬─────┘ └─────┬─────┘ └─────┬─────┘
                  ↓             ↓             ↓
┌─────────────────────────────────────────────────────────────────┐
│              Step 2: 平台专属 Skill                              │
│              风格适配 + 审核规避 + 算法优化 + 格式规范              │
└─────────────────────────────────┬───────────────────────────────┘
                                  ↓
┌─────────────────────────────────────────────────────────────────┐
│              Step 3: 发布                                        │
│              配合 adapters/*.yaml 自动化发布                      │
└─────────────────────────────────────────────────────────────────┘
```

---

## 架构说明

### 混合架构：平台为主 + 通用基座

```
social-publish/
├── WORKFLOW.md                       ← 你在这里
├── copywriting-base/                 ← 通用文案基座（跨平台知识）
│   ├── SKILL.md
│   └── templates/
│       ├── hook-formulas.md
│       ├── viral-patterns.md
│       └── cta-templates.md
├── xiaohongshu/                      ← 小红书专属
│   ├── SKILL.md
│   └── references/
├── twitter/                          ← Twitter/X 专属
│   ├── SKILL.md
│   └── references/
├── douyin/                           ← 抖音专属
│   ├── SKILL.md
│   └── references/
└── bilibili/                         ← B站专属
    ├── SKILL.md
    └── references/
```

**设计原则：**
1. **平台 skill 自包含** — 发小红书只需加载 `xiaohongshu/SKILL.md`
2. **通用知识不重复** — `copywriting-base/` 提供文案原子能力，平台 skill 标注"建议配合 copywriting-base 使用"
3. **风控规则深嵌平台** — 每个平台的 `references/content-rules.md` 包含该平台特有的审核规则
4. **与 adapters 互补** — Skills = AI 内容策略（智能层），Adapters = CDP 自动化操作（机械层）

## 使用方式

### 单平台发布

加载目标平台 skill 即可完成内容创作：

```
# 只发小红书
加载: xiaohongshu/SKILL.md
输入: 选题 + 素材
输出: 符合小红书规范的完整图文/视频文案
```

### 跨平台分发（推荐）

先用 copywriting-base 生成核心文案，再用各平台 skill 适配：

```
Step 1: copywriting-base → 核心文案 + 标题 + CTA
Step 2: parallel
  ├── xiaohongshu/SKILL.md → 小红书版本
  ├── twitter/SKILL.md     → Twitter 版本
  └── douyin/SKILL.md      → 抖音版本
```

### 与 video-skills 联动

视频内容创作可串联 video-skills 和 social-publish：

```
video-skills → 视频成片
      ↓
social-publish/douyin → 抖音文案 + 标签 + 封面文案
social-publish/bilibili → B站标题 + 简介 + 分区标签
```

## Step 1: copywriting-base（通用文案基座）

**职责：** 提供跨平台通用的文案能力

| 能力 | 说明 |
|------|------|
| 标题公式 | 数字型、疑问型、反差型、恐惧型、利益型 |
| Hook 开场 | 前 3 秒 / 前 2 行黄金法则 |
| 正文框架 | AIDA、PAS、BAB 结构 |
| CTA 话术 | 关注、点赞、收藏、转发、评论引导 |
| 病毒因子 | 情绪触发、社交货币、实用价值、故事性 |
| 双语规范 | 中英文适配（非翻译） |

**输出：** 平台无关的核心文案素材，供下游平台 skill 消费。

---

## Step 2: 平台专属 Skill

每个平台 skill 包含：

| 模块 | 说明 |
|------|------|
| 文案风格 | 平台特有的语调、用词习惯 |
| 格式规范 | 图文/视频尺寸、字数限制 |
| 标题规则 | 平台特有的标题写法 |
| 标签策略 | hashtag / 话题标签选择 |
| 发布时间 | 该平台最优发布时段 |
| 审核规避 | 禁用词、敏感领域、限流规则 |
| 算法适配 | 推荐算法权重、流量池机制 |

### 平台对照

| 平台 | 主要形式 | 核心指标 | 风格关键词 |
|------|----------|----------|------------|
| 小红书 | 图文 + 短视频 | CES 分数、完读率 | 种草、姐妹分享、emoji |
| Twitter | 文字 + Thread | 回复率、引用率 | 犀利、观点驱动、简洁 |
| 抖音 | 短视频 | 完播率、互动率 | 口语化、前3秒、信息密度 |
| B站 | 中视频 + 短视频 | 播放+弹幕+投币 | 二次元、技术、一键三连 |

---

## Step 3: 发布

内容创作完成后，使用 `adapters/*.yaml` 完成自动化发布：

```
AI（Skill）生成文案 → CDP（Adapter）自动发布
```

| Adapter | 平台 | 操作 |
|---------|------|------|
| `adapters/xiaohongshu.yaml` | 小红书 | 图文/视频发布 |
| `adapters/x.yaml` | Twitter/X | 推文/Thread 发布 |
| `adapters/douyin.yaml` | 抖音 | 视频发布 |
| `adapters/bilibili.yaml` | B站 | 视频投稿 |

---

## 质量检查清单

发布前自动验证：

- [ ] 无平台禁用词
- [ ] 标题字数符合平台限制
- [ ] 标签数量在推荐范围内
- [ ] 图片/视频尺寸符合规范
- [ ] CTA 存在且合规（无站外引流违规）
- [ ] 发布时间在最优窗口内
